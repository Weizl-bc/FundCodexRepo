# 各维度具体落地方案

------

## 3.1 历史走势

### 目标

判断趋势、强弱、拐点。

### 可计算指标

- MA5、MA10、MA20、MA60
- RSI
- MACD
- KDJ
- 布林带
- 近 5 日、10 日、20 日收益率
- 斜率 slope

### 打分思路

例如总分 100 分拆成：

- MA 多头排列：25 分
- RSI 在合理强势区间：15 分
- MACD 金叉/红柱放大：20 分
- 近 5/10/20 日收益趋势向上：20 分
- 是否突破关键压力位：20 分

### 示例规则

```
若 MA5 > MA10 > MA20，则 +25
若 RSI 在 55~70，则 +15
若 MACD > 0 且连续增强，则 +20
若近10日涨幅 > 近20日均值，则 +20
若突破近30日高点，则 +20
```

------

## 3.2 近期大形势

### 目标

判断当前宏观环境是否支持该资产。

### 可采集信息

- 美联储加息/降息预期
- CPI、PPI
- GDP
- 失业率
- 地缘冲突
- 大宗商品价格
- 汇率变化

### 打分思路

- 宏观利多：70~100
- 中性：40~69
- 宏观承压：0~39

### 实现建议

这个维度适合用 **规则 + NLP摘要**，不要一开始全靠模型。

例如：

- 降息预期增强：+20
- 通胀回落：+20
- 地缘风险升级：-20
- 美债利率快速上行：-20

------

## 3.3 估值走势

### 目标

判断当前买入是否贵。

### 可计算指标

- PE
- PB
- PS
- 股债性价比
- 历史估值百分位
- 行业估值中位数偏离

### 打分思路

这个维度非常适合做规则化。

例如：

- 估值分位 < 20%：90~100 分

- 20%~40%：70~89 分

- 40%~60%：50~69 分

- 60%~80%：30~49 分

- > 80%：0~29 分

### 补充

如果是主题基金、QDII，有时基金本身估值不容易直接拿到，那就用其跟踪指数或核心持仓的估值替代。

------

## 3.4 市场情况

### 目标

判断整体市场活跃度和赚钱效应。

### 指标

- 上涨家数 / 下跌家数
- 涨停数 / 跌停数
- 成交额
- 换手率
- 板块轮动速度
- 指数强弱分化程度

### 打分思路

- 普涨 + 放量：高分
- 缩量 + 普跌：低分
- 极端分化：中低分

### 示例

```
上涨家数 > 下跌家数 1.5 倍：+25
两市成交额明显放大：+25
核心板块持续活跃：+25
赚钱效应扩散：+25
```

------

## 3.5 资金流向

### 目标

判断有没有真金白银推动。

### 数据

- 北向资金净流入
- 主力资金净流入
- ETF 申购赎回
- 板块资金流向
- 行业资金净流入持续天数

### 打分思路

- 连续流入高分
- 单日脉冲中等
- 连续流出低分

### 示例

```
北向资金连续3日净流入：+30
目标行业主力资金净流入：+30
ETF申购明显增加：+20
资金流入扩散而非集中一日：+20
```

------

## 3.6 波动率

### 目标

判断风险是否过大。

### 指标

- 年化波动率
- ATR
- VIX / 中国版恐慌指标
- 最大回撤
- 近 5 日振幅

### 打分思路

这里和其他维度相反，不是越大越好。

- 波动平稳可控：高分
- 波动过大：低分
- 极端恐慌：很低分

### 示例

```
近10日波动率低于历史中位数：80分以上
近10日波动率高于历史80分位：40分以下
极端波动：20分以下
```

------

## 3.7 全球市场联动

### 目标

判断外部市场传导。

### 数据

- 纳指、标普、道指
- 恒生科技
- 日经
- 美债收益率
- 美元指数 DXY
- 原油、黄金
- 人民币汇率

### 适用场景

- QDII 基金
- 科技类基金
- 港股基金
- 新能源 / 黄金 / 原油相关基金

### 打分思路

根据目标基金类型动态选指标。

例如纳指基金：

- 隔夜纳指上涨：+30
- 美债收益率回落：+20
- 科技龙头财报利好：+20
- 美元过强压制风险资产：-20

------

## 3.8 市场情绪指数

### 目标

判断市场是否过热或过冷。

### 数据

- Fear & Greed Index
- 热点词热度
- 涨停情绪
- 社区讨论热度
- 搜索指数

### 打分方式

这里建议做成 **双向修正因子**。

不是情绪越高越好：

- 极度恐慌但基本面稳定：反而可能加分
- 极度贪婪：反而减分

### 示例

```
情绪处于中性偏冷：高分
情绪极端贪婪：减分
情绪极端恐慌但资金回流：加分
```

------

## 3.9 经济周期

### 目标

判断当前资产风格偏向。

### 数据

- PMI
- 社融
- 工业增加值
- 消费数据
- 房地产数据
- 利率周期

### 方式

这个维度不适合高频变动，建议 **周级 / 月级更新**。

### 用法

- 复苏期：成长、顺周期加分
- 衰退期：红利、债券、防御加分
- 滞涨期：黄金、资源、防御加分

------

## 3.10 政策因素

### 目标

判断政策催化。

### 数据来源

- 国常会
- 央行
- 证监会
- 财政政策
- 行业专项扶持政策

### 打分思路

- 明确利好、且直指目标行业：高分
- 模糊表述：中分
- 监管趋严：低分

### 示例

```
若政策明确支持 AI / 半导体 / 新能源，相关基金 +20~40
若政策打压某领域，相关基金 -20~-40
```

------

## 3.11 舆情 / 新闻

### 目标

提取突发事件与短期情绪冲击。

### 数据来源

- 财经新闻
- 公告
- 论坛
- 社交媒体
- 专家观点

### 注意

这个维度不要直接决定买卖，只能作为修正项。

# NLP 具体方案

你已经写了“具体方案：NLP”，那我帮你补成真正能落地的版本。

------

## 一、NLP在系统里的角色

NLP 不应该直接替代全部决策，而应该负责：

### 1. 新闻分类

把新闻分成：

- 宏观类
- 政策类
- 行业类
- 公司类
- 风险事件类

### 2. 情绪判断

判断新闻是：

- 利多
- 中性
- 利空

### 3. 热点提取

提取关键词：

- AI
- 降息
- 通胀
- 芯片
- 地缘冲突
- 新能源
- 港股科技

### 4. 影响对象识别

识别这条新闻影响什么：

- 纳斯达克
- 半导体
- 医药
- 消费
- 黄金
- 债券

### 5. 影响强度评分

例如 0~1：

- 0.2 轻微影响
- 0.5 中等影响
- 0.8 强影响

------

## 二、NLP处理流程

### 第一步：抓取文本

来源：

- 财经新闻站点
- 政策公告
- 社交媒体摘要
- 用户手动上传截图 OCR 后文本化

### 第二步：清洗文本

- 去广告
- 去模板话术
- 去重复新闻
- 标准化时间和机构名

### 第三步：文本分类

分类到：

- 宏观
- 政策
- 行业
- 企业
- 风险事件

### 第四步：情绪分析

输出：

- sentiment = positive / neutral / negative
- score = -1 ~ 1

### 第五步：实体识别

识别：

- 国家
- 政策机构
- 行业
- 指数
- 公司
- 人物

### 第六步：事件抽取

例如抽成结构化结果：

```
{
  "eventType": "policy",
  "title": "美联储释放降息信号",
  "sentiment": "positive",
  "score": 0.72,
  "targets": ["纳斯达克", "科技股", "成长基金"],
  "impactLevel": 0.83,
  "eventTime": "2026-03-09 08:00:00"
}
```

### 第七步：写入数据库

供后续决策引擎使用。
